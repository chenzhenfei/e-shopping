#消息服务平台

主要是处理：邮件消息，短信消息，微信公众号消息的同一公共的消息处理同意平台

步骤：

     1、消息服务平台作为消费者定义需要的消息报文格式，定义需要的接口类型
        消息保温格式一般为JSON 格式：{
                                  "head"：{
                                    "interfacetype":'email'
                                  },
                                  "content":{
                                    "email":916927902@qq.com
                                   }
                                 }
     2、生产者按照消费者定义的接口类型封装消息报文
     3、生产者将消息报文发送到消息中间件中，完成消息的推送
     4、消息服务平台按照监听消息中间件消息来进行消费，通过消息中间件来完成整个系统的流程
     5、通过消息中间件将系统件调用进行了解耦，并且解决了系统件调用其他因素影响的不及时影响问题，这些都会
        通过消息中间件的缓存机制解决，针对消费失败的情况，消息中间件会有自动补偿机制，主要是需要考虑幂等问题，防止重复消费机制发生
